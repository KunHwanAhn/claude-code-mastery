# 코드 스타일 규칙

- 들여쓰기: 스페이스 2칸
- 작성 언어: 영어(코드 표준 준수)
- 변수명: 카멜 케이스 (camelCase)
- 함수명: 카멜 케이스 (camelCase)
- 클래스명: 파스칼 케이스 (PascalCase)
- 상수명: 대문자와 언더스코어 (UPPER_SNAKE_CASE)
- 줄 끝: 세미콜론(;) 사용
- 최대 줄 길이: prettier & ES Lint 설정 파일에 따름. 없다면 150자로 제한
- 공백: 연산자 주변과 콤마 뒤에 한 칸의 공백 사용
- 주석: 코드와 관련된 설명은 한국어로 작성
- 테스트 코드: 테스트 파일은 `__tests__` 폴더에 위치시키고, 파일명은 `*.test.ts` 형식으로 작성. __tests__ 폴더는 테스트 코드 작성 대상 파일과 같은 디렉토리에 위치시킴
- ESLint와 Prettier 설정 파일이 존재하는 경우, 해당 설정을 우선적으로 적용하여 코드 스타일을 유지할 것. 설정 파일이 없는 경우, 위의 규칙을 따를 것
